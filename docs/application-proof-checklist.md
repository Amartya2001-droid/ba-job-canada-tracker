# Application Proof Checklist

Use this before sending any Business Analyst application.

## Required Proof Links

- GitHub repository link
- Frontend dashboard path or hosted URL
- Wait times dashboard preview: `assets/screenshots/wait-times-dashboard-preview.svg`
- Access coverage dashboard preview: `assets/screenshots/access-coverage-dashboard-preview.svg`
- Final Power BI PDF once exported: `assets/screenshots/healthcare-ba-portfolio.pdf`

## What To Mention

- Healthcare is the chosen industry focus.
- The work uses Canadian public data, not random practice datasets.
- SQL/SQLite was used to create repeatable report outputs.
- Power BI-ready visuals were planned from the start.
- Ontario was included after resolving a source-label normalization issue.

## One-Line Project Proof

`I built two healthcare-focused Business Analyst portfolio projects using Canadian public datasets, SQL/SQLite report generation, and dashboard-ready visual outputs for wait-time and access-coverage analysis.`

## Send Only If

- The role asks for at least 70% of the tools or responsibilities you can show.
- The cover note references healthcare, reporting, SQL, dashboards, or stakeholder analysis.
- The application tracker has the company, role, link, stack match, and follow-up date.
