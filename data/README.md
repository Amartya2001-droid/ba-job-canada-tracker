# Data Workspace

This folder keeps the repo organized while you work through data collection and cleaning.

## Structure

- `raw/` for original downloaded files
- `processed/` for cleaned exports used in analysis

## Rules

- Keep original source files unchanged inside `raw/`.
- Save cleaned or merged outputs inside `processed/`.
- Add a short note in the relevant project `data-sources.md` each time you download or refresh a dataset.
- If a file is too large or noisy for GitHub, keep it outside the repo and note the location in the project file.
